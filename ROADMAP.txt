================================================================================
  MONERO-RUST WASM: Web-First Monero Wallet Library - Implementation Roadmap
================================================================================

Project Goal: Build a new Monero wallet library in Rust designed for WebAssembly
from the ground up, with native platform support as a secondary consideration.
This is a greenfield implementation prioritizing WASM constraints and modern
async patterns.

Core Principles:
  - WASM-first architecture (native support in other crates)
  - Pure Rust, no_std compatible where possible
  - Async-first design (no blocking operations, no threads)
  - Environment-agnostic (storage, time, networking abstracted)
  - Leverages monero-serai for cryptographic primitives
  - Exposes clean, minimal API for Flutter/Dart integration

================================================================================
SUCCESS CRITERIA
================================================================================

The project will be considered successful when:

1. A user can create or restore a Monero wallet in a browser (Flutter web)
2. The wallet can connect to a remote Monero node and scan the blockchain
3. Balance is accurately calculated and displayed
4. User can send transactions that are accepted by the network
5. The wallet can be encrypted, saved, and loaded across sessions
6. The browser UI remains responsive during all operations
7. The same codebase compiles for native platforms (iOS, Android, desktop)
8. Comprehensive tests pass on all platforms
9. Documentation is sufficient for third-party integration
10. No critical security vulnerabilities are present

================================================================================
NOTES & CONSIDERATIONS
================================================================================

- This is a from-scratch implementation, not a port of existing code
- WASM constraints are embraced as design principles, not worked around
- Async/await is used throughout; no blocking operations
- All platform-specific code is isolated behind traits
- The library is designed to be embedded, not standalone
- Security is paramount: no shortcuts on cryptography or key handling
- Performance in WASM is secondary to correctness and security
- The API should be minimal and easy to use from Dart/Flutter
- All design decisions prioritize web compatibility first

Key Architectural Decisions Made:
----------------------------------

1. **Rinf Framework for Flutter Integration**
   - Original plan: Direct wasm-bindgen with manual JS/Dart glue
   - Actual: Rinf framework with signal-based communication
   - Benefits: Automatic binding generation, typed request/response, cleaner API
   - Location: dart/monero_extension/

2. **monero-serai as Core Dependency**
   - Forked/mirrored at rust/monero-serai-mirror/
   - Provides all cryptographic primitives
   - Handles transaction construction, scanning, RingCT
   - Comprehensive and well-tested

3. **Test-Driven Development with Real Data**
   - Extensive test suite with real stagenet data
   - "honked bagpipe" test wallet with known outputs
   - Mock RPC infrastructure for deterministic testing
   - Tests demonstrate capabilities without full wallet integration
